== UX smells Data ==
-- Smell: Aesthetic Inconsistency (ID 8)
- Description:
  Aesthetic Mismatch Smell occurs when an app’s visual design, including colors, typography, or themes, does not align with user expectations or significantly deviate from app domain.
- Example:
  A finance app uses bright neon colors and playful fonts, making it feel more like a gaming app rather than a professional financial tool.
- Pseudocode:
  IF (app_theme NOT align_with app_domain) OR
     (app_font NOT align_with app_domain) OR
     (app_color NOT align_with app_domain)
  THEN flag_as("Aesthetic Inconsistency")

-- Smell: Styling Overload (ID 9)
- Description:
  The interface uses excessive or unoptimized effects (e.g., multiple layered shadows, elaborate 3D transitions, or continuous animations) that might distract users from core tasks.
- Example:
  An app uses heavy shadow effects and unnecessary animations on menu transitions, slowing responsiveness.
- Pseudocode:
  IF (overuse_of_animations OR overuse_of_effects) AND (not appropriate for app_domain)
  THEN flag_as("Styling Overload")

-- Smell: Inconvenient Reachability (ID 18)
- Description:
  Neglected Reachability occurs when user interface elements that are frequently needed are placed in positions that require uncomfortable hand movements or adjustments.
- Example:
  A mobile banking app puts the transfer button in the top-right corner, forcing awkward grips and causing mis-taps.
- Pseudocode:
  FOR each frequently_used_element in UI_elements:
    IF element.position.y < (screen_height * 0.5)
    THEN flag_as("Inconvenient Reachability")

-- Smell: Uninspired Stimulus (ID 23)
- Description:
  The app's lack of diversity and creativity, failing to captivate or maintain user interest.
- Example:
  A game app uses the same puzzle format in every level, without introducing new challenges or variations, quickly becoming tedious.
- Pseudocode:
  FOR each UI IN app_UIs:
    IF UI_layout_is_repetitive AND app_domain_requires_creative_stimuli
    THEN flag_as("Uninspired Stimulus")

-- Smell: Cluttered Controls (ID 34)
- Description:
  This occurs when numerous or tightly packed controls clutter a single interface layer.
- Example:
  A form screen shows many small buttons, text fields, and toggles in the same view, making it hard to tap accurately.
- Pseudocode:
  FOR each UI_LAYER in app_screens:
    // 1. Collect visible interactive elements
    CONTROLS = all visible elements of type
               {TextField, Button, Checkbox, Switch,
                RadioButton, ToggleChip, Dropdown, FAB}
    total_controls = COUNT(CONTROLS)

    // 2. Group by visual row/cluster (same horizontal band)
    ROWS = group CONTROLS by y_position band (e.g., 15–20% height windows)
    max_row_density = MAX over ROWS of COUNT(controls_in_row)

    // 3. Flag conditions using UI thresholds
    IF (total_controls >= 8) OR
       (max_row_density >= 4) THEN
      flag_as("Cluttered Controls")

-- Smell: Hidden Features (ID 36)
- Description:
  Essential app features are obscured within complex menus, leading to underuse and frequent user inquiries for navigation help.
- Pseudocode:
  FOR each essential_feature in GUI:
    IF menu_depth_level >= 2
    THEN flag_as("Hidden Features")
